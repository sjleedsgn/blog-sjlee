예전부터 직접 나만의 블로그를 운영해 보고싶었다. 그래서 찾아본 툴로는 예전에는 지킬을 찾아봤었고, 최근에는 Webflow나 Framer로 찾아보고 시도 했었다. 지킬은 테마를 받고 커스텀 할때마다, 완전히 원하는 디자인을 구현하기 힘든 느낌을 받았었다. 시행착오가 크다보니 그런 것 같았다. Webflow나 Framer는 결과물은 나왔지만, 기능적인 측면이나 디테일이 2% 만족스럽지 않았고 그를 핑계로 매번 글을 쓰는건 생각도 못 했다.


대 AI시대에 들어선 지금 클로드 코드와 함께 Next.js+Vercel 로 블로그를 구현했다. 결과는 단순했다. 쉬워보였던 것과 마찬가지로 쉬웠고, 하루가 채 걸리지 않아 최초 배포까지 완료했다. (Phase1 기준, 디벨롭은 별개의 얘기)

## 블로그 PRD
개인기록을 올릴 수 있는 블로그를 만든다.
블로그의 용도는 기록용도 및 포트폴리오 이다.

**목표**

* 마크다운으로 글을 작성하면, 글목록에서 확인할 수 있다.
* 간단히 나를 소개할 수 있다.
* 이력서를 다운받을 수 있다.

**주요 기능**

1. 글 목록 일부(3~5개)조회 / 전체 조회
2. 글 작성은 작성한 md 파일을 프로젝트에 추가하는 형식으로 한다.
3. 이력서 다운로드

**비기능 요구사항**

1. 반응형: 모바일 지원한다.
2. 성능: 페이지 로딩 목표치 (데스크탑 95점 이내, 모바일 85점 이내)
3. SEO: 검색 노출 (SEO: sitemap.xml 생성, robots.txt 설정, og 메타태그 적용, Lighthouse SEO 점수 90점 이상)
4. 접근성: 고려 여부. (WCAG AA 
키보드만으로 모든 페이지 탐색 가능
이미지에 alt 텍스트 적용
텍스트-배경 명도 대비 4.5:1 이상
Lighthouse Accessibility 점수 90점 이상)

**제외범위 정의 (무엇을 하지 않는지)**

1. 댓글 기능 없음
2. 회원가입/로그인 없음
3. 관리자 CMS 없음 (파일 기반으로만 관리)

**성공 지표**

1. 배포 완료 + 월 1회 이상 포스팅
2. 배포 후 1개월 내 월 방문자 3명 달성
3. Google Search Console 등록 및 주요 페이지 색인 완료

**마일스톤**

1. Phase 1: 글 목록 + 상세 페이지 + 소개 페이지 + 배포
2. Phase 2: 이력서 다운로드
3. Phase 3: SEO 최적화 + 접근성 + 방문자수 지표 측정

**기술 스택**

`Next.js`   `Tailwind CSS`   `Vercel`

## 디자인 프로세스

클로드 디자인으로 초안을 받았다. -vibe: 미니멀/절제된 -color: 차가운 뉴트럴(그레이/슬레이트) 스타일로 요청했다. 아래 사진은 클로드 코드에게 피그마에 그려달라고해서 받은 화면이다. 초안으로 받은 디자인은 만족스러웠다. 쓸데없는 프레임들로 간격이 잡혀 있어서 간단한 수정은 필요했다. 


![클로드 디자인 초안](/images/posts/Frame20.png)

여기서 초안을 먼저 클로드코드에게 구현해달라고해서 로컬로 보기 시작했다. 반응형 화면, 컴포넌트 및 디자인 가이드를 추가로 작성해서 클로드 코드에게 먹여가며 디벨롭 시켰다.

## Claude Code 후기

클로드 디자인으로 초안을 받았고, 클로드 코드에게 핸드오프해 피그마에 그려진 초안을 받았다. 직접 figma로 디자인 디테일을 잡고나서 Claude Code에게 구현을 맡겼다. 놀라운 건 생각보다 많은 것이 정확하게 구현됐다는 점이다. 조금씩 끊어서 맡겼어야 했었을거다, 구현이 제대로 되지않은 것 들은 다시 맡겨서 완성했다.

**클로드 코드를 디자이너가 써본 후기**
1. 평소 갖고 있던 AI의 FOMO 일부 해소
2. 정말 재밌는 작업이였다는 것. 생각한대로 모두 구현이 가능했다. (코드 퀄리티는... 잘 모르겠다)

예전부터 나만의 블로그를 직접 운영해보고 싶었다. 처음엔 Jekyll을 찾아봤고, 최근엔 Webflow나 Framer를 시도해봤다. Jekyll은 테마를 받아 커스터마이징할 때마다 원하는 디자인을 구현하기가 쉽지 않았다. 시행착오가 많다 보니 그런 것 같았다. Webflow와 Framer는 결과물은 나왔지만 기능이나 디테일에서 2%쯤 아쉬움이 남았고, 그걸 핑계로 매번 글 쓰는 건 엄두도 못 냈다.


AI 시대에 들어선 지금, Claude Code와 함께 Next.js + Vercel로 블로그를 만들었다. 결과는 단순했다. 쉬워 보였던 것만큼 실제로도 쉬웠고, 하루가 채 걸리지 않아 최초 배포까지 완료했다. (Phase 1 기준)

## 블로그 PRD
개인 기록을 올릴 수 있는 블로그를 만든다. 용도는 기록과 포트폴리오, 두 가지다.

**목표**

* 마크다운으로 글을 작성하면, 글목록에서 확인할 수 있다.
* 나를 간단히 소개할 수 있다.
* 이력서를 다운받을 수 있다.

**주요 기능**

1. 글 목록 일부(3~5개)조회 / 전체 조회
2. 글 작성은 .md 파일을 프로젝트에 추가하는 방식으로 한다.
3. 이력서 다운로드

**비기능 요구사항**

1. 반응형: 모바일 지원
2. 성능: 데스크탑 95점, 모바일 85점 이내
3. SEO: sitemap.xml, robots.txt, OG 메타태그, Lighthouse SEO 90점 이상
4. 접근성: WCAG AA 수준 (키보드 탐색, alt 텍스트, 명도 대비 4.5:1 이상, Lighthouse 접근성 90점 이상)

**제외범위 정의 (무엇을 하지 않는지)**

1. 댓글 기능 없음
2. 회원가입/로그인 없음
3. 관리자 CMS 없음 (파일 기반으로만 관리)

**성공 지표**

1. 배포 완료 + 월 1회 이상 포스팅
2. 배포 후 1개월 내 방문자 3명 달성
3. Google Search Console 등록 및 주요 페이지 색인 완료

**마일스톤**

1. Phase 1: 글 목록 + 상세 페이지 + 소개 페이지 + 배포
2. Phase 2: 이력서 다운로드
3. Phase 3: SEO 최적화 + 접근성 + 방문자수 지표 측정

**기술 스택**

`Next.js`   `Tailwind CSS`   `Vercel`

## 디자인 프로세스

Claude 디자인으로 초안을 받았다. 미니멀하고 절제된 분위기에, 차가운 뉴트럴(그레이/슬레이트) 계열 색상으로 요청했다. 아래 이미지는 Claude Code에게 Figma에 그려달라고 요청해 받은 화면이다. 초안 디자인은 꽤 마음에 들었다. 다만 불필요한 프레임들로 간격이 어색하게 잡혀 있어서 간단한 수정이 필요했다.


![클로드 디자인 초안](/images/posts/Frame20.png)

초안을 받고 나서 바로 Claude Code에게 구현을 맡겨 로컬로 확인하기 시작했다. 이후 반응형 화면, 컴포넌트, 디자인 가이드를 직접 작성해 Claude Code에 넘기며 디벨롭해 나갔다.

## Claude Code 후기

Claude 디자인으로 초안을 받고, Claude Code에 핸드오프해 Figma 초안을 그렸다. 이후 직접 Figma에서 디자인 디테일을 다듬은 뒤 Claude Code에게 구현을 맡겼다. 놀라웠던 건, 생각보다 많은 것들이 정확하게 구현됐다는 점이다. 끊어서 맡기지 않고 한꺼번에 넘겨서 일부는 제대로 되지 않았지만, 다시 요청해 마무리할 수 있었다.

**클로드 코드를 디자이너가 써본 후기**
1. 평소 갖고 있던 AI에 대한 FOMO가 어느 정도 해소됐다.
2. 정말 재밌는 작업이었다. 생각한 것들이 대부분 구현됐다. (코드 퀄리티는... 솔직히 잘 모르겠다)


**작업하다 생긴 생각**
1. 업무 프로세스에 어떻게 적용할 수 있을까?
2. 기존 디자인 시스템을 연결해서 활용한다면, 가능성이 어디까지일까?

당분간은 디자인 작업을 할 때마다 이번 경험이 계속 영향을 줄 것 같다. 어떻게 일상적인 업무에 녹여낼 수 있을지 계속 생각해봐야겠다.
